## Step 4: Compiling your source code

A compiler does following:
- Does your C++ code follow the rules of the C++ language?
- Translates your code into machine language instructions (stored in an intermediate file called an **object file**)


| Feature            | Object File                                                                                                                                   | Executable File                                     |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------- |
| File formats       | `.o` `.obj`                                                                                                                                   | `.exe` `.out`                                       |
| Directly Runnable? | No                                                                                                                                            | Yes                                                 |
| Content            | Incomplete machine code chunks                                                                                                                | Fully linked machine code and libraries             |
| Scope              | Represents and single code file                                                                                                               | Unified, self-contained program                     |
| Addresses          | Relocatable (placeholder addresses)                                                                                                           | Fixed absolute or relative virtual memory addresses |

Compiling to an executable format with g++:
```
g++ main.cpp -o a.out
```

Compiling without linking and then linking:
```
g++ -c foo.cpp  -o foo.o
g++ -c bar.cpp  -o bar.o
g++ -c main.cpp -o main.o

g++ foo.o bar.o main.o -o software
```

## Step 5: Linking object files and libraries and creating the desired output file

After the compiler has successfully finished, another program called the **linker** runs to combine all of the object files and produce the desired output file. This process is called **linking**. The process a linker follows:
- Read each object files generated by the compiler and make sure they are valid
- Ensure all cross-file dependencies are resolved properly. For example, if you define something in one .cpp file, and then use it in a different .cpp file, the linker connects the two together. If the linker is unable to connect a reference to something with its definition, you'll get a linker error and the process will abort.

## Definitions

- **Library files**: precompiled code that have been "packaged up" for reuse in other programs
- **Building**: process of converting source code files into an executable that can be run (specific executable produced is called a **build**)
- **Debugging**: process of finding and fixing program errors
